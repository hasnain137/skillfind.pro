## üìù Documentation Best Practices

### When Writing Docs

1. **Be Clear:** Use simple language
2. **Be Specific:** Include exact commands
3. **Be Complete:** Don't assume knowledge
4. **Be Visual:** Use formatting, examples
5. **Be Current:** Update regularly

### When Reading Docs

1. **Skim First:** Read headings and summaries
2. **Use Search:** Ctrl+F is your friend
3. **Try Examples:** Test code snippets
4. **Ask Questions:** Open issues for unclear parts
5. **Contribute Back:** Fix errors you find
